### 평가 항목 (Criteria)

각 항목은 1-10점으로 평가합니다.

#### 1. Claude Code / AI 도구 활용 능력 (claudeCodeProficiency)
- **가중치:** 25%
- **우수 (8-10):** Claude Code로 실제 프로젝트 빌드, 멀티에이전트 활용, 바이브코딩 숙련
- **보통 (5-7):** Cursor, Copilot 등 AI 코딩 도구 사용 경험, Claude Code는 미경험
- **미달 (1-4):** AI 도구 사용 경험 없음

#### 2. 빠른 프로토타이핑 능력 (rapidPrototyping)
- **가중치:** 25%
- **우수 (8-10):** 3일 내 PoC 시연 가능, 다수 MVP 경험, 스타트업 환경
- **보통 (5-7):** 프로젝트 경험은 있으나 속도보다 완성도 중심
- **미달 (1-4):** 프로토타이핑 경험 부족, 대기업 워터폴 방식만 경험

#### 3. 지원 동기 — 간절함과 적합도 (motivation)
- **가중치:** 20%
- **Tally 질문:** "왜 함께 일하고 싶나요? 지원 배경을 알려주세요."
- **우수 (8-10):** 조쉬의 비전/콘텐츠에 공감, AX에 대한 구체적 관심, "왜 이 일인지" 명확, 간절함
- **보통 (5-7):** AI 관심사는 있으나 Joshua/AX에 대한 이해 부족
- **미달 (1-4):** 동기 불분명, 성의 없는 답변, 포지션과 무관

#### 4. 일과 비전 — 업무 이해도 (dailyWorkVision)
- **가중치:** 15%
- **Tally 질문:** "채용이 되신다면, 어떤 업무와 일과로 하루를 보낼 것 같으신가요?"
- **우수 (8-10):** Daily Scrum + PoC 사이클 + 클라이언트 소통 언급, 현실적인 일과, Self-motivated
- **보통 (5-7):** 개발 중심 일과만 언급, 컨설팅/클라이언트 측면 부족
- **미달 (1-4):** 비현실적이거나 관련 없는 답변, "시키는 대로" 식 답변

#### 5. 기술 스택 적합성 (techStackFit)
- **가중치:** 15%
- **우수 (8-10):** Supabase, PostgreSQL 실무 경험, AI/LLM API 연동 경험
- **보통 (5-7):** 유사 기술 경험 (Firebase, MySQL), 웹 개발 가능
- **미달 (1-4):** 개발 경험 부족 (단, 태도와 학습력이 뛰어나면 가점)

### 점수 기준 (Verdict Thresholds)

- **strong** (강력 추천): 총점 80 이상
- **maybe** (검토 필요): 총점 55-79
- **pass** (불합격): 총점 55 미만

### 총점 계산

가중 평균 방식:
- claudeCodeProficiency × 0.25
- rapidPrototyping × 0.25
- motivation × 0.20
- dailyWorkVision × 0.15
- techStackFit × 0.15

총점 = (각 점수 × 가중치)의 합 × 10

### 레드 플래그 (Dealbreakers)

- 답변이 모두 1-2줄로 성의 없음 → pass
- 이력서가 포지션과 완전히 무관 → pass
- 복붙으로 보이는 답변 → pass
- 풀타임 출퇴근 불가 → pass

### 보너스 포인트

- 조쉬의 유튜브/뉴스레터/콘텐츠 언급 → +5
- 창업 경험 또는 FDE 경험 → +5
- 글로벌(영어) 커뮤니케이션 가능 → +3
